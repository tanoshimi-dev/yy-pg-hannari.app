## 1️⃣ Webサイトを公開するって？

ローカルPC（自分のパソコン）で作ったWebサイトは、自分だけしか見られません。
**レンタルサーバー** にファイルをアップロードすると、世界中の人がアクセスできるようになります。

### 📊 ローカルと公開サーバーの違い

```
ローカル環境（開発中）                    レンタルサーバー（公開後）
┌───────────────────────┐           ┌───────────────────────┐
│ 自分のパソコン          │           │ インターネット上のPC    │
│                       │  FTPで     │                       │
│  📁 htdocs/           │ アップ     │  📁 public_html/      │
│    ├── index.html     │ ────────→ │    ├── index.html     │
│    ├── style.css      │ ロード     │    ├── style.css      │
│    └── script.js      │           │    └── script.js      │
│                       │           │                       │
│  http://localhost/    │           │  https://example.com/ │
│  → 自分だけ見られる     │           │  → 世界中から見られる！  │
└───────────────────────┘           └───────────────────────┘
```

### 🔧 公開に必要なもの

| 必要なもの | 説明 | 費用の目安 |
|-----------|------|----------|
| レンタルサーバー | ファイルを置く場所 | 月500〜1,500円 |
| ドメイン | Webサイトの住所（URL） | 年1,000〜3,000円 |
| FTPクライアント | ファイルを転送するソフト | 無料 |

### 📝 レンタルサーバー契約の流れ

| ステップ | やること |
|---------|--------|

```

  サーバー接続情報

```

> **⚠️ この情報は絶対に他人に見せないでください！**
> 悪用されると、サイトを改ざんされる危険があります。

## 3️⃣ FTPってなに？

### 📦 FTP ＝ ファイルを転送する仕組み

FTPは **File Transfer Protocol**（ファイル・トランスファー・プロトコル）の略です。
自分のパソコンからサーバーにファイルを送る（アップロード）ための仕組みです。

```
自分のパソコン               インターネット               サーバー
┌─────────┐                                    ┌─────────┐
│ 📁 ファイル│  ───── FTPで転送 ──────────────→  │ 📁 ファイル│
│         │                                    │         │
│  index.html │  アップロード（送る）              │  index.html │
│  style.css  │  ──────────────→                │  style.css  │
│             │                                │             │
│  index.html │  ダウンロード（受け取る）           │  index.html │
│             │  ←──────────────                │             │
└─────────┘                                    └─────────┘
```

### 🔒 FTPとSFTP / FTPSの違い

| プロトコル | 安全性 | 説明 |
|-----------|-------|------|
| FTP | ⚠️ 低い | データが暗号化されない。学習用にはOK |
| FTPS | ✅ 高い | SSL/TLSで暗号化。多くのサーバーが対応 |
| SFTP | ✅ 高い | SSHで暗号化。最も安全 |

> **💡 本番では FTPS または SFTP を使おう！**
> FileZillaはすべてに対応しています。

## 4️⃣ FileZilla をインストールしよう！

FileZilla（ファイルジラ）は、世界で最も使われている**無料のFTPクライアント**です。

### 📥 ダウンロード & インストール

| ステップ | やること |
|---------|--------|
| STEP 1 | https://filezilla-project.org/ にアクセス |
| STEP 2 | 「Download FileZilla Client」をクリック |
| STEP 3 | 無料版の「Download」を選ぶ（有料版は不要！） |
| STEP 4 | ダウンロードしたファイルでインストール |
| STEP 5 | 起動すると、FileZillaの画面が開く！ |

> **⚠️ 注意**
> ダウンロードページで「FileZilla Pro」（有料版）をすすめられることがありますが、
> 無料の「FileZilla Client」で十分です。

### 📊 FileZilla の画面構成

```
┌──────────────────────────────────────────────────────┐
│  接続バー（ホスト名・ユーザー名・パスワード・ポート）        │
├──────────────────────┬───────────────────────────────┤
│                      │                               │
│  📂 ローカルサイト      │  📂 リモートサイト               │
│  （自分のパソコン）     │  （サーバー）                    │
│                      │                               │
│  ├── Desktop/        │  ├── public_html/              │
│  ├── Documents/      │  │   ├── index.html            │
│  ├── my-app/         │  │   ├── style.css             │
│  │   ├── index.html  │  │   └── images/               │
│  │   ├── style.css   │  ├── mail/                     │
│  │   └── script.js   │  └── log/                      │
│                      │                               │
│  左側 = 自分のPC      │  右側 = サーバーの中              │
│                      │                               │
├──────────────────────┴───────────────────────────────┤
│  転送状況（アップロード/ダウンロードの進行状況）             │
└──────────────────────────────────────────────────────┘
```

- **左半分** → 自分のパソコンのファイル
- **右半分** → サーバーのファイル
- 左から右にドラッグ → **アップロード**
- 右から左にドラッグ → **ダウンロード**

## 5️⃣ サーバーに接続しよう！

### 📝 クイック接続（かんたん！）

FileZilla上部の接続バーに情報を入力します：

| 項目 | 入力する内容 | 例 |
|-----|------------|-----|
| ホスト | FTPホスト名 | ftp.example.com |
| ユーザー名 | FTPユーザー名 | username |
| パスワード | FTPパスワード | ******** |
| ポート | 21（FTP）または 22（SFTP） | 21 |

「クイック接続」ボタンをクリック！

### 🔖 サイトマネージャー（おすすめ！）

毎回情報を入力しなくていいように、接続先を保存できます。

| ステップ | やること |
|---------|--------|
| STEP 1 | メニュー「ファイル」→「サイトマネージャー」を開く |
| STEP 2 | 「新しいサイト」をクリック |
| STEP 3 | サイト名をつける（例：「ロリポップ」） |
| STEP 4 | 接続情報を入力 |
| STEP 5 | 「接続」をクリック |

### 📋 サイトマネージャーの設定例

| 項目 | 設定値 |
|-----|--------|
| プロトコル | FTP - ファイル転送プロトコル |
| ホスト | ftp.example.com |
| ポート | （空欄でOK） |
| 暗号化 | 明示的な FTP over TLS が必要（推奨） |
| ログオンタイプ | 通常 |
| ユーザー | username |
| パスワード | ******** |

### ✅ 接続成功の確認

接続に成功すると：
- 右側にサーバーのフォルダ一覧が表示される
- 下のステータスバーに「ディレクトリ一覧の表示成功」と出る
- `public_html` や `www` フォルダが見える

## 6️⃣ ファイルをアップロードしよう！

### 📁 アップロード先のフォルダ

レンタルサーバーによって、Webサイトのファイルを置くフォルダが異なります：

| レンタルサーバー | 公開フォルダ |
|--------------|-----------|
| ロリポップ! | /home/users/○/username/ |
| エックスサーバー | /home/username/public_html/ |
| さくら | /home/username/www/ |
| ConoHa WING | /public_html/ |

### 📤 アップロードの手順

| ステップ | やること |
|---------|--------|
| STEP 1 | FileZillaでサーバーに接続する |
| STEP 2 | **右側**で公開フォルダ（public_htmlなど）を開く |
| STEP 3 | **左側**でアップロードしたいファイルのフォルダを開く |
| STEP 4 | 左側のファイルを選択して、右側にドラッグ&ドロップ！ |
| STEP 5 | 下の転送状況で「転送完了」と表示されたらOK |

### ⚠️ アップロード時の注意点

| 注意 | 説明 |
|-----|------|
| ファイル名は半角英数字 | 日本語のファイル名は避ける（文字化けの原因） |
| 拡張子を確認 | `.html` `.php` `.css` `.js` になっているか |
| フォルダ構造を合わせる | ローカルと同じ階層構造でアップロードする |
| index.html を忘れない | トップページのファイル名は `index.html` か `index.php` |

### 🎯 アップロードの実践例

```
ローカルのフォルダ構成          サーバーにアップロード後
my-app/                      public_html/
├── index.html        →      ├── index.html
├── style.css         →      ├── style.css
├── script.js         →      ├── script.js
└── images/           →      └── images/
    ├── logo.png      →          ├── logo.png
    └── photo.jpg     →          └── photo.jpg
```

アップロード後、ブラウザで `https://あなたのドメイン/` にアクセスして確認！

## 7️⃣ ドメインとURLの仕組み

### 🏠 ドメイン ＝ Webサイトの住所

```
https://www.example.com/about/index.html
  ↑        ↑              ↑        ↑
プロトコル  ドメイン名      フォルダ   ファイル名
(通信方法)  (住所)         (部屋番号) (部屋の中の書類)
```

### 🌐 ドメインの種類

| 種類 | 例 | 説明 |
|-----|-----|------|
| 初期ドメイン | username.server.jp | サーバー契約時に無料でもらえる |
| 独自ドメイン | example.com | 自分だけの住所。年額1,000円〜 |
| サブドメイン | blog.example.com | 独自ドメインの一部を分けて使う |

> **💡 まずは初期ドメインで十分！**
> 学習段階では、サーバーからもらえる初期ドメインでOKです。
> 本格的に公開するときに独自ドメインを取得しましょう。

### 📝 独自ドメインを取得するには

| サービス | 特徴 |
|---------|------|
| お名前.com | 国内最大手。セールが多い |
| ムームードメイン | ロリポップ!と連携しやすい |
| Google Domains | シンプルで分かりやすい |

## 8️⃣ ファイルの更新（再アップロード）

Webサイトを修正したら、変更したファイルだけを再アップロードします。

### 🔄 更新の手順

| ステップ | やること |
|---------|--------|
| STEP 1 | ローカルでファイルを修正・保存する |
| STEP 2 | FileZillaでサーバーに接続 |
| STEP 3 | 修正したファイルをドラッグ&ドロップで上書きアップロード |
| STEP 4 | 「上書きしますか？」→「はい」を選ぶ |
| STEP 5 | ブラウザでページを再読み込み（Ctrl+F5）して確認 |

### 💡 効率アップのコツ

| コツ | 説明 |
|-----|------|
| 変更ファイルだけアップ | 全ファイルを毎回アップする必要はない |
| キャッシュクリア | 更新が反映されないときはブラウザのキャッシュをクリア（Ctrl+F5） |
| バックアップ | 大きな変更前に、サーバーのファイルをダウンロードして保存しておく |

## 9️⃣ 公開前チェックリスト

Webサイトを公開する前に、以下をチェックしましょう！

### ✅ チェック項目

```
□ HTMLに文法エラーがないか
□ リンク切れ（404エラー）がないか
□ 画像が正しく表示されているか
□ スマホでも見やすいか（レスポンシブ対応）
□ 文字化けしていないか
□ パスワードやAPIキーがコードに含まれていないか
□ 不要なテストファイルがアップされていないか
□ ファビコン（タブのアイコン）が設定されているか
```

### ⚠️ セキュリティチェック

```
□ PHPのエラー表示がオフになっているか
   → ini_set('display_errors', 0);
□ フォームにhtmlspecialchars()を使っているか
□ パスワードをコードに直書きしていないか
□ .htaccessで不要なフォルダへのアクセスを制限しているか
□ テスト用のphpinfo()ファイルを削除したか
```

## 🔟 トラブルシューティング

| 問題 | 原因 | 解決方法 |
|-----|------|---------|
| 接続できない | 接続情報が間違っている | ホスト名・ユーザー名・パスワードを再確認 |
| 接続できない | パッシブモードの問題 | FileZillaの設定で「パッシブ」に変更 |
| 403 Forbidden | パーミッション（権限）の問題 | ファイル:644、フォルダ:755 に設定 |
| 404 Not Found | ファイルが正しい場所にない | public_html直下にアップされているか確認 |
| 500 Internal Server Error | PHPのエラー | エラーログを確認、コードを修正 |
| ページが古いまま | ブラウザのキャッシュ | Ctrl+F5 でスーパーリロード |
| 画像が表示されない | パスが違う / ファイル名の大文字小文字 | サーバーは大文字小文字を区別するので注意 |
| 文字化けする | 文字コードが違う | ファイルをUTF-8で保存、HTMLにcharset指定 |

### 🔐 パーミッション（権限）の設定

サーバー上のファイルには「誰がアクセスできるか」の設定があります。

| 対象 | 推奨値 | 説明 |
|-----|--------|------|
| HTMLファイル | 644 | 所有者:読み書き / 他:読み取りのみ |
| PHPファイル | 644 | 同上 |
| フォルダ | 755 | 所有者:全権限 / 他:読み取り+実行 |
| 書き込み用フォルダ | 707 | PHPから書き込みが必要な場合 |
| .htaccess | 604 | 設定ファイル。慎重に |

FileZillaでファイルを右クリック →「ファイルのパーミッション」で変更できます。
