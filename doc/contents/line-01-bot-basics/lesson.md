## 3️⃣ プロバイダーを作ろう！

ログインできたら、まず「プロバイダー」を作ります。

### 🏢 プロバイダーってなに？

プロバイダーは「ボットの持ち主」を表すものです。
個人の名前やチーム名を入れます。

```
プロバイダーの例：
  🏢 「田中たろうの開発室」
  🏢 「はんなりチーム」
  🏢 「5年1組プログラミング部」
```

### 📝 プロバイダー作成手順

| ステップ | やること |
|---------|--------|
| STEP 1 | LINE Developersのトップページで「コンソール」をクリック |
| STEP 2 | 「新規プロバイダー作成」をクリック |
| STEP 3 | プロバイダー名を入力（例：「〇〇の開発室」） |
| STEP 4 | 「作成」をクリック |

> **💡 ポイント**
> - プロバイダー名はあとから変更できます
> - 1つのプロバイダーの中に、複数のボット（チャネル）を作れます

## 4️⃣ Messaging APIチャネルを作ろう！

プロバイダーができたら、次は「チャネル」を作ります。
チャネルが「1つのボット」に対応します。

### 📱 チャネルってなに？

```
プロバイダー（持ち主）
  ├── チャネル1 = 日記ボット 📔
  ├── チャネル2 = お小遣い帳ボット 💰
  └── チャネル3 = クイズボット ❓
```

1つのチャネル ＝ 1つのボット と覚えよう！

### 📝 チャネル作成手順

| ステップ | やること | 詳細 |
|---------|--------|------|
| STEP 1 | プロバイダーのページで「新規チャネル作成」をクリック | |
| STEP 2 | チャネルの種類で「Messaging API」を選ぶ | ※「LINEログイン」ではないので注意！ |
| STEP 3 | 必要な情報を入力する | 下の表を参考にしよう |
| STEP 4 | 利用規約に同意して「作成」をクリック | |

### 📋 チャネル作成時に入力する情報

| 項目 | 入力例 | 説明 |
|-----|-------|------|
| チャネルの種類 | Messaging API | 必ずこれを選ぶ！ |
| プロバイダー | （先ほど作ったもの） | 自動で選ばれているはず |
| チャネルアイコン | 好きな画像 | ボットのアイコンになる（あとで変更可） |
| チャネル名 | 日記ボット | LINEの友だちリストに表示される名前 |
| チャネル説明 | 日記を記録するボットです | ボットの説明文 |
| 大業種 | 個人 | 個人開発なら「個人」でOK |
| 小業種 | 個人（その他） | |
| メールアドレス | （自分のメール） | 通知が届くアドレス |

## 5️⃣ 大事な設定を確認しよう！

チャネルができたら、ボットを動かすために必要な情報を確認・設定します。

### 🔑 チャネルシークレット

チャネルの「パスワード」のようなものです。

| 手順 | やること |
|-----|--------|
| 1 | チャネルの設定ページを開く |
| 2 | 「チャネル基本設定」タブをクリック |
| 3 | 「チャネルシークレット」の欄を確認 |
| 4 | メモしておこう（あとでプログラムに使います） |

### 🔑 チャネルアクセストークン

ボットがLINEにメッセージを送るときに使う「合言葉」です。

| 手順 | やること |
|-----|--------|
| 1 | 「Messaging API設定」タブをクリック |
| 2 | ページ下部の「チャネルアクセストークン」まで移動 |
| 3 | 「発行」ボタンをクリック |
| 4 | 表示された長い文字列をコピーしてメモ！ |

> **⚠️ 超重要！セキュリティに注意**
>
> チャネルシークレットとアクセストークンは **絶対に他の人に見せないでください！**
> これがあれば、誰でもあなたのボットを操作できてしまいます。
>
> - SNSに貼り付けない
> - GitHubの公開リポジトリにアップしない
> - 友だちに教えない
>
> もし漏れてしまったら、すぐに「再発行」ボタンで新しいものを作りましょう。

### 📊 2つのキーの役割まとめ

```
チャネルシークレット（Channel Secret）
  → LINEから届いたメッセージが「本物かどうか」を確認するためのカギ
  → 署名検証（しょめいけんしょう）に使う

チャネルアクセストークン（Channel Access Token）
  → ボットからLINEにメッセージを送るためのカギ
  → APIリクエストの認証に使う
```

## 6️⃣ Messaging APIの設定をしよう！

### 🔧 応答設定

ボットの基本的な動きを設定します。

| 手順 | やること |
|-----|--------|
| 1 | 「Messaging API設定」タブを開く |
| 2 | 「応答メッセージ」の「編集」リンクをクリック |
| 3 | LINE Official Account Managerが開く |
| 4 | 以下のように設定する |

### 📋 おすすめの応答設定

| 設定項目 | 値 | 理由 |
|---------|---|------|
| 応答メッセージ | オフ | 自動応答を使わず、プログラムで返信するため |
| あいさつメッセージ | オン or オフ | お好みで。友だち追加時のメッセージ |
| Webhook | オン | **必ずオンにする！** プログラムにメッセージを届けるために必要 |

> **💡 Webhookとは？**
>
> ユーザーがボットにメッセージを送ったとき、LINEがあなたのプログラムに
> 「メッセージが届いたよ！」と知らせてくれる仕組みです。
>
> ```
> 📱 ユーザーがメッセージ送信
>   ↓
> 🔔 LINEが「Webhook URL」に通知を送る
>   ↓
> ⚙️ あなたのプログラムが受け取って処理する
>   ↓
> 📱 ボットが返信する
> ```

### 🌐 Webhook URLの設定

Webhook URLは、プログラム（GASなど）を作ったあとに設定します。

| 手順 | やること |
|-----|--------|
| 1 | 「Messaging API設定」タブを開く |
| 2 | 「Webhook URL」の欄を探す |
| 3 | GASのWebアプリURLを貼り付ける（あとで設定！） |
| 4 | 「Webhookの利用」をオンにする |
| 5 | 「検証」ボタンをクリックして「成功」と出ればOK！ |

> **📝 GASのWebアプリURLの取得方法**
>
> GASのエディタで：
> 1. 「デプロイ」→「新しいデプロイ」をクリック
> 2. 種類で「ウェブアプリ」を選ぶ
> 3. 「アクセスできるユーザー」を「全員」に設定
> 4. 「デプロイ」をクリック
> 5. 表示されるURLをコピーして、Webhook URLに貼り付ける

## 8️⃣ GASでボットのプログラムを書く準備

チャネル設定が終わったら、いよいよプログラムを書きます！

### 📝 GASの準備手順

| ステップ | やること |
|---------|--------|
| STEP 1 | Googleドライブで新しいスプレッドシートを作る |
| STEP 2 | 「拡張機能」→「Apps Script」を開く |
| STEP 3 | 最初から書いてあるコードを全部消す |
| STEP 4 | 以下のテンプレートコードを貼り付ける |

### 💻 最小限のボットコード（テンプレート）

```javascript
// ===== 設定 =====
// ここにチャネルアクセストークンを貼り付ける
const ACCESS_TOKEN = 'ここにあなたのアクセストークンを貼り付け';

// ===== メイン処理 =====
// LINEからメッセージが届いたら、この関数が自動で動く
function doPost(e) {
  // 届いたデータを読み取る
  const json = JSON.parse(e.postData.contents);
  const event = json.events[0];

  // メッセージイベント以外は無視
  if (event.type !== 'message' || event.message.type !== 'text') {
    return;
  }

  // ユーザーが送ったメッセージ
  const userMessage = event.message.text;

  // 返信メッセージを作る
  const replyMessage = 'あなたは「' + userMessage + '」と言いました！';

  // LINEに返信する
  replyToLine(event.replyToken, replyMessage);
}

// ===== LINEに返信する関数 =====
function replyToLine(replyToken, message) {
  UrlFetchApp.fetch('https://api.line.me/v2/bot/message/reply', {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + ACCESS_TOKEN,
    },
    payload: JSON.stringify({
      replyToken: replyToken,
      messages: [{ type: 'text', text: message }],
    }),
  });
}
```

### ▶ デプロイして動かす

| ステップ | やること |
|---------|--------|
| STEP 1 | コードを貼り付けて、ACCESS_TOKENを自分のものに変える |
| STEP 2 | 「デプロイ」→「新しいデプロイ」 |
| STEP 3 | 種類を「ウェブアプリ」に設定 |
| STEP 4 | 「アクセスできるユーザー」を「全員」にする |
| STEP 5 | 「デプロイ」をクリック |
| STEP 6 | 表示されたURLをコピー |
| STEP 7 | LINE DevelopersのWebhook URLに貼り付けて保存 |
| STEP 8 | LINEでボットにメッセージを送ってみよう！ |

## 9️⃣ トラブルシューティング

### ❓ よくある問題と解決方法

| 問題 | 原因 | 解決方法 |
|-----|------|---------|
| ボットが返信しない | Webhookがオフ | Messaging API設定でWebhookをオンにする |
| ボットが返信しない | Webhook URLが未設定 | GASのWebアプリURLを設定する |
| ボットが返信しない | アクセストークンが間違っている | トークンを再発行してコードを更新 |
| 「権限がありません」エラー | GASの権限未承認 | 実行時に表示される権限を「許可」する |
| 「Webhook URLの検証に失敗」 | GASが正しくデプロイされていない | 新しいデプロイを作り直す |
| コードを変更したのに反映されない | デプロイが古いまま | 「デプロイを管理」→「新しいバージョン」で更新 |

> **💡 デプロイの更新について**
>
> GASのコードを修正したら、**必ず新しいデプロイを作る**必要があります！
> 1. 「デプロイ」→「デプロイを管理」
> 2. 鉛筆アイコン（編集）をクリック
> 3. バージョンを「新バージョン」に変更
> 4. 「デプロイ」をクリック
>
> ※ URLは変わらないので、Webhook URLの再設定は不要です。
